Project 2 expands EduTrack to seven screens by adding a Study Session screen and a public Leaderboard, and connects the app to the outside world across all four technical pillars below.

Cloud integration

Internet data

Reflection & learning journey

Lab 1 → Project 2

Looking back at Lab 1, the app barely did anything — a fixed layout with no logic at all. Each lab added exactly one capability on top of the last: interaction in Lab 2, visual polish in Lab 3, navigation and shared state in Lab 4, and persistence in Lab 5. By the time Project 1 arrived, those pieces combined naturally into a working 5-screen app, and Project 2 simply extended the same patterns outward — to the cloud, to the internet, and to the phone's hardware.

The biggest challenge throughout was environment and configuration issues — package name mismatches, Gradle/KSP version conflicts, and file-location errors caused far more build failures than any actual Compose UI code did. Working through those taught me to read error logs more carefully and to understand why a fix works, not just copy it.

In terms of SDG impact, EduTrack stays small and personal by design — it doesn't try to replace a school's LMS, but it gives an individual student a lightweight way to plan subjects, see their own progress, and build a habit of focused study sessions, which is exactly the kind of low-barrier tool SDG 4 (Quality Education) calls for.
